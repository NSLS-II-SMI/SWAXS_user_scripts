# Per-Group User-Folder Analysis - Small Groups (SMI / SWAXS)

Beamline: NSLS-II SMI (12-ID), SWAXS. Scope: classify scientific use-cases and
data-acquisition patterns for the NEW per-group folder convention (each PI/institution
its own folder). Classification pass only; **no fixes proposed**.

Detector legend: `pil2M` = SAXS, `pil900KW`/`pil300KW` = WAXS, `rayonix` = MAXS,
`pin_diode`/`pdcurrent2` = transmission/I0 photodiode, `xbpm2`/`xbpm3` = beam-position
monitors (`.sumX`), `amptek` = energy-dispersive fluorescence MCA, `OAV_writing` =
on-axis-video frame capture. Heaters: `LThermal` (Linkam MFS/THMS), `ls` (Lakeshore).

Pattern grading recap:
- **MODERN** = one Bluesky run per logical sample via `@bpp.run_decorator(md={...})` +
  `@bpp.stage_decorator(dets)` around `inner()`, data via `bps.trigger_and_read(dets+[...])`,
  context (T/energy/angle/filename) recorded as Signals/devices, md dict not `RE.md`.
- **LEGACY** = nested `for` loops each firing `bp.count`/`bp.scan`/`bp.rel_scan`/`bp.grid_scan`
  (a run per data point), global mutable `sample_id(...)`/`project_set(...)` for filenames,
  context read by `.get()`/`.value`/`.position` into filename strings, hard-coded
  name/coordinate lists in the plan body.

## File: `Commissioning/bounce_down_mirror.py`  (~604 lines, ~22 plans)
- **2. Group/purpose:** Beamline staff. **XRR (X-ray reflectivity) via a bounce-down
  mirror (`bdm`)** - the headline new infrastructure here. Bends the beam down by `bdm.th`
  to reach a fixed liquid/solid surface at negative effective angles, with extensive
  ROI/geometry helpers, knife-edge alignment, attenuator auto-selection, and direct-beam
  vs reflected-beam tracking.
- **3. Use-cases:** XRR (solid & liquid surfaces), GISAXS/GIWAXS alignment, liquid-surface
  reflectivity, knife-edge height alignment, commissioning/geometry calibration,
  attenuation management.
- **3b. Detectors:** `pil2M` (SAXS, primary - direct/reflected ROIs `roi1/2/3`),
  `pil900KW` (WAXS variant).
- **4. Pattern: MODERN (the XRR run-plans) + procedural helpers.** The four `run_xrr_*`
  /`xrr_scan_*` plans each = one `@stage_decorator`+`@run_decorator(md=...)` run sweeping
  angle, recording an `incident_angle` Signal and a `target_file_name` Signal via
  `trigger_and_read`. Many *helper* routines (`scan_knife_edge`, `check_*`, `move_bdm_sample`,
  alignment) are interactive `bp.rel_scan`/`count` + `ps()` peak-finding (alignment-style,
  not data runs).
  ```python
  @bpp.stage_decorator([det])
  @bpp.run_decorator(md={'file_name':'{target_file_name}','scan_name':'XRR_bounce_down'})
  def inner():
      for angle in angles:
          yield from bps.mv(bdm.th, ai0+angle)
          yield from smi.setReflectedBeamROI(total_angle=-angle, technique="gisaxs", ...)
          s.put(...); incident_angle.put(angle)
          yield from bps.trigger_and_read([det]+[s,incident_angle])
  ```
- **5. Notable:** `bdm` (bounce-down mirror) device; `smi.setReflectedBeamROI(...)` /
  `setDirectBeamROI()` / `modeAlignment()`; geometric `move_bdm_sample()` (tan(2*alpha)*L
  y-offset); angle-dependent attenuator ladder `att_selection_8keV()` (Mo foils att2_1/2/3);
  `peizo_th_correction()` linear theta-backlash/scale fit; `ps.cen` derivative peak-finding;
  records `incident_angle` as a proper Signal (good practice).
- **6. Intent:** Reusable bounce-down-mirror XRR + surface-alignment toolkit (solid &
  liquid), one of the most genuinely modern data-run patterns in this corpus.

# CROSS-FOLDER SYNTHESIS

- **Distinct scientific archetypes across these small groups:** (1) tender-energy resonant
  GIWAXS/NEXAFS edge scans (Cl/S/K/P/Ca/Fe/Ti/etc., ~2.1-2.9 keV) - NIST Richter & LBL Su;
  (2) in-situ blade-coating / meniscus-deposition drying kinetics (LBL Su, syringe pump +
  Thorlabs stage); (3) hard-X-ray transmission/capillary SWAXS bars (UVA Cai, SBU Takeuchi);
  (4) operando/in-situ microbeam raster grain-mapping (Cornell Singer/Das); (5) temperature-
  ramp annealing micro-scans (UVA Cai, NIST Aiello - Linkam/Lakeshore); (6) in-situ
  tensile & 3D-printing (UVA Cai legacy); (7) XRR via bounce-down mirror, incl. liquid
  surfaces (Commissioning); (8) calibration/commissioning utilities (AgBehenate SDD/center,
  attenuator ladders, micro-raster GUI templates); (9) a stray XPCS plan (LBL Su `xpcs_2025_1`).
- **Detector convention is uniform and clean:** `pil900KW` = WAXS (priority), `pil2M` = SAXS
  (dropped whenever `waxs.arc` blocks the SAXS path, via `[pil900KW] if arc<~15 else [pil2M,pil900KW]`);
  `pil300KW` only in Richter transmission; `pin_diode`/`pdcurrent2` + `xbpm2/3` for I0;
  `amptek` for fluorescence (Richter); `OAV_writing` to log optical position. **No Rayonix/MAXS**
  anywhere in this small-group set.
- **Genuinely MODERN (target-style) files:** `nist/richter/Cl_nexafs.py` (best - single run,
  up/down sweep, **filename templated from recorded fields** not `.get()` strings),
  `nist/richter/p_nexafs.py` (clean single-run NEXAFS, proper imports + averaging-time),
  and the `run_xrr_*`/`xrr_scan_*` plans in `Commissioning/bounce_down_mirror.py` (one run
  per sweep, `incident_angle` recorded as a Signal). `Commissioning/microlistscan.py`
  (GUI-generated) also exemplifies the intended single-run grid/list-scan-per-region idiom.
- **MODERN-envelope-but-still-legacy-interior (the dominant "thin migration"):**
  `Commissioning/s_edge_grazing.py`, `nist/richter/multi_giwaxs_tender.py`, and the 2025
  plans in `LBL/30-user-Su.py` / `nist/richter/30-user-Richter-202502.py`. They have the
  correct `stage_decorator`+`run_decorator`+`trigger_and_read` skeleton, but keep hard-coded
  `names[]`/`x_piezo[]` bars, `xbpm2.sumX.get()`/`.position` formatted into filename strings,
  a throwaway `target_file_name` Signal, and the literal unsubstituted md placeholder
  `'{target_file_name}'`. `multi_giwaxs_tender.py` additionally has 4 copy-pasted inner1-4
  run blocks + commented-out duplicates (mid-edit cruft) - functional but not clean.
- **STILL FULLY LEGACY despite being purpose-named/refactored:** `LBL/bladecoating.py`
  (`count`/`scan`+`sample_id`), `UVA/Cai/capillary_transmission_saxs.py` (`count`+`sample_id`
  + `db[-1].table()` reach-back to compute transmission), `UVA/Cai/temperature.py`
  (`rel_scan` + **`RE.md['sample_name']` string filenames** - the explicitly-discouraged
  channel), `nist/aiello/micro_tempstage.py` (`rel_scan`+`sample_id`). Renaming the file did
  NOT modernize the acquisition pattern in these four.
- **Whole-folder maturity ranking:** NIST Richter is the most advanced (two clean
  NEXAFS exemplars + envelope GIWAXS); Commissioning is mixed but contains the strongest
  reusable modern infra (XRR/BDM); LBL Su is mid-migration (envelope GIWAXS + Linkam +
  XPCS, but legacy blade-coating); Cornell/Singer, SBU/Takeuchi, and UVA/Cai remain
  effectively legacy (grid/count/scan + `sample_id`), even in their newest files.
- **Reusable Commissioning infrastructure worth promoting to shared library:**
  (a) `bounce_down_mirror.py` - a full XRR/BDM toolkit (`smi.setReflectedBeamROI`,
  geometric `move_bdm_sample`, angle-dependent attenuator ladder `att_selection_8keV`,
  `peizo_th_correction` theta-scale fit, knife-edge `ps.cen` alignment, `incident_angle`
  Signal); (b) `microlistscan.py` - the `swaxs-beam-image` GUI's grid/list-scan templates
  (single-run-per-region, positions auto-appended, filename-via-Signal) - effectively the
  reference for micro-mapping; (c) `AGB_scan.py` - AgBehenate SDD/beam-center calibration;
  (d) `attenuation_testing.py` - attenuator transmission ladder.
- **Recurring damage-mitigation / robustness idioms (cross-group, reusable):** fresh-spot
  tiling during edge sweeps (`piezo.x = xs - counter*30um`; NIST, Commissioning); beam-loss
  recovery (`if xbpm2.sumX.get()<50: re-move energy`); WAXS-move retry-on-`FailedStatus`
  loops; `try/except` alignment with failures logged to `RE.md['misaligned_samples']`
  (Takeuchi) or triple-retry + `RE.clear_suspenders()` (Cai temperature). These are mature
  operational patterns living inside otherwise-legacy plan bodies.
- **Pervasive cross-cutting legacy smells:** hard-coded parallel `names[]/x_piezo[]/...`
  bars guarded by `assert len(...)`; global mutable `sample_id()`/`project_set()`/`proposal_id()`;
  context (T, energy, SDD, position, transmission, elapsed time) computed via
  `.get()`/`.position`/`time.time()`/`db[-1].table()` and **baked into filename strings**
  rather than recorded as stream fields; `RE.md['sample_name']` used directly (Cai temperature,
  Singer). Even the modern envelopes inherit the string-filename and hard-coded-bar habits.
- **Heater-device fragmentation:** `LThermal` (Linkam) used by NIST Aiello, LBL Su,
  Cai `temperature.py`; `ls` (Lakeshore) used by Cai `capillary_transmission_saxs.py` and
  Cai/30-user. The two Cai refactored files disagree on heater device - a portability hazard.
  Where T is included, the better files put `LThermal.temperature_current`/`ls.input_A_celsius`
  into `dets` (in-stream) even while filenames still duplicate it as a string.
- **Do the purpose-named refactored files represent the target best-practice style?**
  **Partially.** `Cl_nexafs.py`/`p_nexafs.py` (NEXAFS) and `microlistscan.py`/`bounce_down_mirror.py`
  (Commissioning) DO - single run per logical experiment, `trigger_and_read`, context as
  Signals/devices, md as dict, filename templated from recorded fields (Cl_nexafs is the
  gold reference). But `bladecoating.py`, `capillary_transmission_saxs.py`, `temperature.py`,
  and `micro_tempstage.py` are refactored in *name and organization only* - they remain
  `count`/`scan`/`rel_scan` + `sample_id`/`RE.md` legacy internally. **Conclusion:**
  carving small purpose-named files out of the giant `30-user-*.py` run-books is real
  progress in discoverability, but the per-event-run + global-filename anti-patterns largely
  survived the move; only the NEXAFS and XRR/micro-mapping files reach the modern target.
